# Форумы

Классический PHP-движок интернет-форума с поддержкой анонимного и зарегистрированного общения, модерации, коммерческих разделов и внутренней платёжной системы. Проект ориентирован на русскоязычную аудиторию.

## Основные возможности

### Форум
- Группы тем (разделы) с коммерческими, некоммерческими и смешанными типами
- Темы и сообщения с постраничной навигацией (50 тем / 50 сообщений на страницу)
- «Топ 50» — горячие темы по последней активности и по популярности
- Закрепление тем в топе раздела и общем топе (платные и бесплатные опции)
- Поднятие («ап») старых тем с ограничениями по частоте
- Рейтинг сообщений и система кармы пользователей
- Голосование за сообщения
- Поиск по форуму
- Загрузка изображений к сообщениям, встраивание видео (YouTube и др.)
- BBCode-разметка с фильтрацией через Jevix

### Пользователи
- Анонимное участие по нику или регистрация с подтверждением
- Авторизация по cookie-сессии
- Личный кабинет: профиль, аватар, избранные темы, тарификация
- Внутренняя почта («пейджер») между пользователями
- Список пользователей
- Восстановление пароля

### Модерация и безопасность
- Типы банов: полный, только авторизация, только чтение, антифлуд
- Блокировка по IP, подсетям и никам
- Скрытие и удаление тем и сообщений
- Фильтр запрещённых слов
- CAPTCHA для анонимных пользователей
- Защита от флуда (`System_Protector`)
- Проверка IP-адреса пользователем (`/auth/`, `/banned/`)
- Режим профилактики (`PREVENTION_MODE`) с заглушкой `offline.php`

### Коммерция и платежи
- Платные пакеты услуг и коммерческие темы
- Поднятие тем в топ за внутреннюю валюту
- Интеграция с Robokassa, WebMoney (WMR), Wallet One (W1)
- Административное управление тарифами и пакетами

### Администрирование
- Панель администратора (`.admin/`)
- Управление разделами, темами, сообщениями, пользователями
- Настройка банов, коммерческих пакетов, объявлений
- Модераторские действия через AJAX-события

## Технологии

| Компонент | Решение |
|-----------|---------|
| Язык | PHP (legacy, PHP 5.x API: `ereg`, `mcrypt`) |
| БД | MySQL через [DbSimple](core/lib/dbsimple/) |
| Шаблоны | Smarty (встроенный парсер в `core/lib/system/parser/`) |

| Кэш | XCache (опционально) |
| Почта | PHPMailer |
| Изображения | PHPThumb |

## Архитектура

Приложение построено на собственном каркасе с convention-based автозагрузкой классов:

```
Folder_ClassName → core/lib/folder/class.php  или  core/modules/folder/class.php
```

Точка входа — `index.php`. Жизненный цикл запроса:

1. Инициализация (`System_Init`) — подключение к БД, разбор URL
2. Авторизация (`Auth_Manager`)
3. Проверка защиты (`System_Protector`) — баны, флуд
4. Перехват POST-событий (`EventInterceptor`) — обработка `event=...`
5. Маршрутизация на модуль по первому сегменту URL (`pages.cfg`)
6. Рендеринг Smarty-шаблона

### Модули

| URL-префикс | Класс | Назначение |
|-------------|-------|------------|
| `/forum/` | `Forum_Manager` | Основной форум |
| `/register/` | `Forum_Registration` | Регистрация |
| `/personal/` | `Forum_Personal` | Личный кабинет |
| `/pager/` | `Forum_Pager` | Личные сообщения |
| `/users/` | `Forum_Userlist` | Список пользователей |
| `/search/` | `Search_Manager` | Поиск |
| `/pages/` | `Pages_Manager` | Статические страницы |
| `/.admin/` | `Forum_Admin` | Админ-панель |
| `/auth/`, `/banned/` | `Forum_Banned` | Проверка блокировок |

POST-действия (создание темы, отправка сообщения, модерация и т.д.) описаны в `core/config/events/` и привязаны к обработчикам `onEvent_*`.

```
forum/
├── index.php              # Точка входа

├── offline.php            # Страница технических работ
├── antibot.php            # Антибот-проверка
├── actions.php            # Внешний информер (legacy)
├── route.php              # Логирование переходов по URL
│
├── core/
│   ├── config/
│   │   ├── base.php           # Основные константы и настройки
│   │   ├── pages.cfg          # Маршрутизация модулей
│   │   ├── admin.cfg.php      # Конфигурация админ-панели
│   │   ├── events/            # Обработчики POST-событий
│   │   ├── filter/            # Списки фильтрации
│   │   ├── lang/              # Языковые строки
│   │   └── system/
│   │       └── database.cfg   # Строка подключения к MySQL
│   │
│   ├── lib/                   # Библиотеки (auth, form, mail, money, system…)
│   ├── modules/               # Бизнес-логика (forum, search, vote, pages)

├── js/                        # JavaScript
├── style/                     # CSS-стили
```

## Требования

- PHP 5.x с расширениями: `mysql`, `mcrypt`, `gd`, `session`
- MySQL 5.x
- Веб-сервер (Apache/Nginx) с поддержкой mod_rewrite или аналогом
- Опционально: XCache для кэширования запросов и информеров

## Установка

1. Скопировать проект в корень веб-сервера.
2. Создать базу данных MySQL и импортировать схему (дамп БД в репозиторий не входит).
3. Настроить подключение в `core/config/system/database.cfg`:

   ```
   mysql://LOGIN:PASSWORD@localhost/FORUM
   ```

4. Создать директории с правами на запись:

   ```bash
   mkdir -p tmp/templates_c tmp/cache upload
   chmod 755 tmp upload
   ```

5. Настроить константы в `core/config/base.php` (лимиты, стоимость услуг, режим профилактики и др.).
6. Настроить веб-сервер: все запросы должны направляться на `index.php` с сохранением path-info для маршрутизации вида `/forum/{group_id}/{theme_id}/`.

## Конфигурация

Ключевые параметры задаются в `core/config/base.php`:

| Константа | Описание |
|-----------|----------|
| `PREVENTION_MODE` | `1` — включить режим технических работ |
| `COMMERCIAL_ON` | Включение коммерческих функций |
| `PER_PAGE` | Количество элементов на странице |
| `USER_VOTE_LEVEL` | Минимальная карма для голосования |
| `MESSAGE_HIDE_RATING` | Рейтинг, ниже которого сообщение скрывается |
| `FRESH_USERS_SECOND_LIMIT` | Ограничение для новых пользователей (сек.) |
| `PROTECT_FLOOD_HITS` | Лимит запросов для антифлуд-защиты |
